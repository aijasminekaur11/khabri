# 📱 Telegram Bot - User Guide

**Simple guide for using the Khabri Fix Bot on your phone**

---

## 🎯 What Can You Do?

- 🔧 **Request fixes** for bugs or issues
- 📊 **Check status** of your requests
- ❓ **Get help** anytime

---

## 📝 How to Use

### 1️⃣ Request a Fix

When something is broken or not working:

**Type in Telegram:**
```
/fix The problem you're seeing
```

**Examples:**

```
/fix Budget alerts not showing real estate news
```

```
/fix Email notifications not coming through
```

```
/fix Login page showing error on mobile
```

```
/fix Competitor tracking missing new listings
```

**What happens next:**

1. ✅ Bot confirms: "Creating fix request..."
2. 🤖 System analyzes the problem
3. 🔧 Creates an automatic fix
4. 📬 You get a notification when it's ready
5. ✅ You review and approve the fix

---

### 2️⃣ Check Status of Your Requests

**Type in Telegram:**
```
/status
```

**You'll see:**
```
📊 Recent Fix Requests

✅ #123 - Budget alerts fixed
   Status: Completed

🔄 #122 - Email notifications
   Status: PR Ready (tap to review)

⏳ #121 - Login page issue
   Status: In Progress
```

**What the symbols mean:**
- ✅ = Fixed and done
- 🔄 = Fix ready for review
- ⏳ = Still working on it

---

### 3️⃣ Get Help

**Type in Telegram:**
```
/help
```

Shows all available commands and examples.

---

## 💡 Tips for Best Results

### ✅ Good Fix Requests

**Be specific:**
```
/fix Budget alerts showing crypto news instead of real estate news
```

**Include what you see:**
```
/fix Email says "error 500" when I try to login
```

**Mention when it happens:**
```
/fix Morning digest not arriving at 8 AM anymore
```

### ❌ Avoid Vague Requests

**Too vague:**
```
/fix Something is broken
```

**Better:**
```
/fix Telegram notifications stopped working today
```

---

## 🔔 Notifications

You'll automatically receive Telegram messages when:

1. ✅ **Fix is ready** - "Your fix is ready for review!"
2. ❌ **Fix needs manual work** - "This needs developer review"
3. 🎉 **Fix is merged** - "Your fix is now live!"

---

## 📱 Reviewing Fixes (Optional)

When you get "Fix ready for review" notification:

1. **Tap the GitHub link** in the message
2. **GitHub mobile app opens** (or browser)
3. **See what changed** in the "Files changed" tab
4. **If it looks good**, tap "Merge pull request"
5. **Confirm** the merge

**Don't worry!** If you're not sure, just wait - someone will review it.

---

## 🆘 Common Questions

### Q: How long does a fix take?

**A:** Usually 5-15 minutes. Complex issues may take longer or need manual review.

### Q: Can I send multiple fix requests?

**A:** Yes! Send as many as you need. Check `/status` to see them all.

### Q: What if the fix doesn't work?

**A:** Just send another `/fix` message explaining what's still wrong.

### Q: Can I cancel a request?

**A:** Yes, just message and we'll close it on GitHub.

### Q: What time can I send requests?

**A:** Anytime! The bot runs 24/7.

---

## 📞 Need Human Help?

If the bot can't help, just send a normal message (no `/fix`) describing the problem and someone will respond.
